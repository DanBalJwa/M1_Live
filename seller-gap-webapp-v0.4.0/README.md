# 셀러 갭 소싱 웹앱 v0.4.0

카테고리를 1차부터 4차까지 선택하면 후보 키워드를 자동으로 만들고, 각 키워드의 네이버 상품·쿠팡 공개 검색정보를 순서대로 표에 채우는 개인용 로컬 웹앱입니다.

## 이번 버전의 핵심 동작

1. 1차 → 2차 → 3차 → 4차 분류를 선택합니다.
2. 돋보기 버튼을 누릅니다.
3. 선택한 세부 카테고리에 등록된 기본 키워드와 네이버·쿠팡 상품명에서 추출한 표현을 합쳐 후보 키워드를 만듭니다.
4. 키워드를 두 개씩 병렬 조사하며 표가 순차적으로 채워집니다.
5. `N 네이버 상품`, `C 쿠팡`, `통합 비교` 탭을 각각 확인합니다.
6. 필요한 조건으로 필터링하고 현재 탭을 CSV로 저장합니다.

## 실행

1. ZIP을 내려받아 압축을 풉니다.
2. 저장소 전체가 아니라 압축을 푼 폴더 안의 `seller-gap-webapp-v0.4.0` 폴더로 들어갑니다.
3. `실행.bat`을 더블클릭합니다.
4. 브라우저가 자동으로 열리지 않으면 `http://127.0.0.1:8000`에 접속합니다.

필요 환경:

- Windows 10/11
- Python 3
- 별도의 Python 패키지 설치는 필요하지 않습니다.

직접 실행:

```bash
python app.py
```

## NAVER API HUB 설정

우측 상단의 `API 설정`에서 Client ID와 Client Secret을 저장합니다.

이 버전에서 필요한 권한:

- **검색어 트렌드**
- **쇼핑 인사이트**
- **검색 API → 쇼핑**

기존 v0.3.1과 달리 네이버 상품 수·가격·쇼핑몰 표본을 조회해야 하므로 `검색 API의 쇼핑` 권한이 추가로 필요합니다. 이미지, 뉴스, 블로그, 웹문서, 오타 변환은 필요하지 않습니다.

`저장 후 연결 테스트`를 누르면 다음을 각각 검사합니다.

- 검색어 트렌드 연결
- 네이버 쇼핑 검색 연결

Client Secret은 로컬의 `data/config.json`에 저장됩니다. 폴더를 타인에게 전달하기 전 이 파일을 삭제하세요.

- 현재 공개 검색화면에서 읽힌 상품 표본

- 최대·평균 리뷰 수
- 로켓·로켓그로스·광고 표본 수
- 표본 기반 경쟁도

### 통합 비교 탭

- 네이버와 쿠팡 중앙가격 차이
- 네이버 해외배송 비율
- 쿠팡 로켓 비율
- 양쪽 경쟁도
- 3개월 증감률
- 단순 규칙 기반 1차 판정

## 중요: 실제값과 추정값 구분

다음은 공개되거나 API가 반환하는 값입니다.

- 네이버 쇼핑 API의 검색결과 수, 상품명, 가격, 판매몰
- 네이버 검색어 트렌드와 쇼핑 인사이트의 상대지수
- 쿠팡 공개 검색화면에 표시된 가격, 리뷰, 로켓·광고 표식

다음은 공개되지 않으므로 제공하지 않습니다.

- 경쟁 판매자의 실제 판매량
- 실제 순매출·순이익
- 광고비·전환율
- 쿠팡의 정확한 전체 판매량

경쟁도는 상품 수, 판매몰 집중도, 로켓·광고 비율, 리뷰 크기 등 공개 표본을 조합한 **프로그램 내부 비교점수**입니다. 플랫폼의 공식 지표가 아닙니다.

## 쿠팡 조회 제한

쿠팡은 자동 조회를 제한할 수 있습니다. 제한될 경우 해당 키워드 행에 오류 원인이 표시되며, 프로그램은 차단 우회·CAPTCHA 우회·프록시 회전을 수행하지 않습니다. 네이버 결과는 정상적으로 계속 표시됩니다.

## 카테고리 추가

기본 카테고리는 `data/categories.json`에 있습니다. 현재는 사입 검토와 관련성이 높은 패션의류, 깔창·신발용품, 작업보조용품, 미용 연습용품 등을 우선 수록했습니다.

항목 구조:

```json
{
  "name": "작업복바지",
  "code": "work-pants",
  "seeds": [
    "남성 작업바지",
    "여름 작업바지",
    "빅사이즈 작업바지"
  ]
}
```

다른 품목을 추가하려면 같은 구조로 1~4차 `children`을 확장하면 됩니다. 숫자 8자리의 네이버 카테고리 코드가 경로에 있으면 쇼핑 인사이트 호출에도 활용합니다.

## 문제 해결

### 검색어 트렌드는 정상인데 네이버 상품이 오류

NAVER API HUB Application에서 `검색 API → 쇼핑`을 추가해야 합니다.

### 표가 늦게 채워짐

키워드마다 네이버 트렌드, 쇼핑 인사이트, 네이버 쇼핑, 쿠팡 검색을 요청합니다. 결과는 전부 끝날 때까지 기다리지 않고 완료되는 순서대로 표시됩니다.

### 쿠팡만 전부 오류

쿠팡 측 접근 제한일 가능성이 큽니다. 잠시 후 재시도하되, 반복 호출하지 말고 쿠팡 검색화면에서 최종 수동 확인하세요.

### 포트가 이미 사용 중이라는 오류

실행 중인 이전 버전의 검은 콘솔 창을 닫은 뒤 다시 실행하세요.
