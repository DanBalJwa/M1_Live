# 셀러 갭 소싱 웹앱 v0.4.1

카테고리를 1차~4차 중 원하는 단계까지만 선택하고 네이버와 쿠팡의 공개 상품 데이터를 각각 분리해 조사하는 로컬 웹앱입니다.

## v0.4.1 수정사항

- 1차, 2차, 3차까지만 선택해도 조사 가능
- 검색어 트렌드는 **네이버 통합검색** 기준으로 명시
- 쇼핑 클릭 추이는 **네이버 통합검색 쇼핑 영역 + 네이버쇼핑** 기준으로 명시
- 존재하지 않는 NAVER API HUB 쇼핑검색 연결 테스트 제거
- API 연결 테스트는 `검색어 트렌드`와 `쇼핑 인사이트`만 수행
- 쿠팡 탭에서 네이버 트렌드 값을 재사용하던 오류 제거
- 쿠팡 검색량·클릭 추이는 공개 API가 없으므로 `제공되지 않음`으로 표시
- 네이버·쿠팡 상품 공개 검색화면 수집 결과를 서로 독립적으로 표시

## 실행

1. 폴더 안의 `run.bat`을 실행합니다.
2. 자동으로 열리지 않으면 브라우저에서 `http://127.0.0.1:8000`을 엽니다.
3. 우측 상단 `API 설정`에서 NAVER API HUB Client ID와 Client Secret을 저장합니다.
4. API Application에는 **검색어 트렌드**와 **쇼핑 인사이트**만 등록하면 됩니다.
5. 1차~4차 카테고리 중 원하는 단계까지만 고른 뒤 돋보기를 누릅니다.

## 데이터 기준

- 네이버 통합검색 추이: NAVER API HUB `/search-trend/v1/search`
- 네이버 쇼핑 클릭 추이: NAVER API HUB `/shopping/v1/category/keywords`
- 네이버·쿠팡 상품 정보: 각 공개 검색화면에서 한 번에 보이는 표본
- 쿠팡 검색량·클릭 추이: 공개 API가 없어 표시하지 않음

두 트렌드 API의 값은 절대 검색량이 아니라 각 조회 결과 안에서 정규화된 상대지수입니다. 서로 다른 API의 숫자를 절대 규모로 직접 비교하면 안 됩니다.

## 제한

네이버·쿠팡이 로그인, CAPTCHA 또는 화면 구조 변경을 적용하면 공개 검색화면 분석이 실패할 수 있습니다. 차단 우회나 대량 자동수집은 하지 않습니다.
